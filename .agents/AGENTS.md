# Tandem-AIO Agent Behavioral Rules & Coding Guidelines

These rules dictate how code should be written, styled, and committed in this repository.

## 1. Coding Style & Understandability
- **Beginner-Friendly:** Write code in a way that is highly accessible so that even a beginner coder can understand what is going on. Prioritize extreme clarity and readability over clever, dense, or complex one-liners.
- **Naming Conventions:** Follow the exact naming conventions already established in the existing codebase. Match the surrounding style perfectly.
- **Pyramid Coding Style:** Lay the groundwork first (basic structure, core types, interfaces, or setup) and then build up gradually to the more complex logic. 

## 2. Commenting Rules
- **Human-like Comments:** Leave comments in a natural, conversational, human method. Avoid using a bunch of trailing dashes, robotic formatting, or overly verbose structural markers. Write explanations as if you are explaining the code to a teammate.

## 3. Git Workflow
- **Frequent Commits:** Periodically stage changes (`git add`) and commit every hundred lines or so (or after small, logical milestones). Do not build up massive changes for a single commit.
- **Human-like Commits:** Write commit messages that sound human (e.g., "added basic flask app with tandem task", "fix dummy wasm fallback"). Keep them concise, simple, and casual. Nothing fancy or too long, and definitely no robotic, overly detailed automated formats.
